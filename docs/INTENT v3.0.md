
# Landbrief 개발 의도 (INTENT.md)

> **버전**: v3.0 (2026-03-01)
> **저장 경로**: `~/landbrief/docs/INTENT.md`
> **목적**: 바이브 코딩 세션에서 AI가 맥락을 잃지 않도록 프로젝트 의도·원칙·범위를 고정하는 문서.
> **정본 규칙**: 코드와 문서가 불일치하면 문서를 수정한다.


---

## 1. 제품 정의

Landbrief는 공공 부동산 데이터를 동일 기간·동일 단위로 정렬·정제하여, 실무자가 즉시 엑셀로 추출해 회사 제출 자료로 활용할 수 있도록 지원하는 부동산 데이터 추출 도구다.

판단 없음. 점수 없음. 예측 없음. 자체 산출 비율 없음. 정렬 + 정제 + 엑셀 출력에 집중한다. 원천 데이터를 정렬해서 제공하고, 계산은 사용자 몫이다.


---

## 2. 절대 원칙

### Do

- 객관적 수치, 통계, 추이를 동일 기간·동일 단위로 정렬하여 제공한다.
- 지표는 "기관명_정식명칭"으로 표기한다(코드도 함께 유지).
- 모든 엑셀에 단위, 면적 기준, 데이터 출처, 집계 방식을 명기한다.

### Don't

- "상승/하락/침체/회복/위험/추천" 같은 해석 문장 금지.
- 점수/등급/신호등/AI 예측/투자판단/리스크 판단 결과 제공 금지.
- 자체 산출 비율(전세가율, 거래회전율 등) 제공 금지.
- 가중평균 등 공시 기준 없는 자체 합산 산식 사용 금지.
- 원천 실거래 전체 CSV 재판매(전체 DB 제공/대량 다운로드) 금지.
- KB 및 민간 데이터 사용 금지.

## 3. 데이터 원칙

### 3.1 데이터 범위

모든 데이터는 공공데이터 기반으로 구성한다. KB부동산 이용약관 제12조④ 영리목적 이용 금지, 제14조①-5 크롤링 금지에 따라 KB 및 민간 데이터는 사용하지 않는다.

### 3.2 데이터 소스

국토교통부 실거래가(DATA_GO_KR), 한국부동산원(REB), 한국은행(ECOS), 행정안전부(MOIS), 국토교통부 통계(MOLIT_STAT), 청약홈(APPLYHOME), 건축물대장(ARCH_HUB, 보강용).

### 3.3 면적 기준

공급평단가 산출: 거래금액 ÷ 공급면적 × 3.3058(평 환산). 공급면적은 건축물대장 API에서 확보한다. 확보 전까지 전용면적 기준으로 제공하며 "전용면적 기준"을 명시한다. 모든 엑셀에 면적 기준을 명기한다.

### 3.4 단지코드

국토부 apt_seq 기반으로 자체 구축한다. KB 단지코드 의존 없음. apt_seq는 매매/임대 간 동일 코드 체계임을 검증 완료(37,289개 단지 공유, 90.6% 일치).

### 3.5 데이터 확보 전략

월별 데이터(최근 36개월): API 자동 수집.

연별 데이터(10년): CSV 수동 다운로드 → PostgreSQL 적재. 실거래가 공개시스템, 통계누리, 행안부 등에서 시군구별 연 단위 집계값을 확보한다.

### 3.6 분양 평단가 산출 정책

원천: 청약홈 APT 주택형별 분양정보(공공데이터포털 #15101047).

산출: 해당 단지에서 세대수가 가장 많은 주택형의 분양최고금액 ÷ 주택공급면적.

제공 필드가 "분양최고금액"만 존재한다(평균/최저 미제공). 이것이 공공 데이터에서 분양가를 확보할 수 있는 유일한 공식 경로다. 엑셀에 "분양최고금액 기준" 및 기준 주택형을 명기한다.

### 3.7 결측 처리 정책

데이터 없는 셀은 "-" 표기한다.

엑셀 하단 고지문: "본 자료는 공공데이터포털·한국은행·한국부동산원 API 제공 데이터 기준으로 작성되었으며, API 미제공 기간 또는 지역의 데이터는 누락될 수 있습니다."

### 3.8 지역 수준 불일치 처리

시군구 데이터 없는 지표는 시도 또는 전국 단위 데이터를 사용한다. 행 레이블에 지역 수준을 명기한다(예: "기준금리 (전국)", "주택가격전망CSI (경기도)"). 전국/시도 대비 순위·백분위 표시는 검토 예정이다.

### 3.9 거래량 집계 우선순위

1차: APT_SALE_DTL / APT_RENT 원천 실거래 건수 집계(Landbrief 자체 산출). 참고: REB_TRADE_VOLUME(R-ONE 집계) 병기.

## 4. Phase 구조

### Phase 1: 회사 파일럿 — CLI 엑셀 출력

목표: 개발지역 인근 정보를 회사 제출용 엑셀로 자동 생성한다.

입력: 시군구 1개 선택. 기간 옵션(월별 36개월, 연별 10년).

```
python generate_excel.py --sgg 11680 --months 36
python generate_excel.py --sgg 11680 --years 10
```

산출물: 엑셀 파일 (4시트). 시트 구성은 섹션 6 참조.

### Phase 2: 웹 기반 추출 SaaS

FastAPI + Jinja2 웹 인터페이스. Kakao Map 기반 시군구 선택. 단지 검색·비교 기능. 좌표 기반 반경 자동 검색. 사용자 인증 및 과금 구조. 유료 로우데이터 시트 제공.

### Phase 3: 리포트 자동화 (기업 요청 기반)

PF 심사, 투자심의, 중개사 대응 리포트 자동 생성. Phase 1~2의 정렬 엔진과 엑셀 엔진을 기반으로 확장한다.

## 6. 엑셀 구성 (Phase 1)

### 입력

시군구 1개 선택(예: 11680 강남구). 화성시 등 구 단위 데이터 미존재 지역은 시 단위로 제공한다.

### 시트 구성: 4시트

#### 시트1: 연별 추이 (10년)

행: 지표, 열: 2016 ~ 2025.

하나의 시군구에 대해 모든 지표를 한 표에 배치한다. 지역 수준이 다른 지표는 행 레이블에 명기한다.

|------|---------|----------|-------------|
| 가격 | REB 매매가격지수 | 시군구 | 12월 값 |
| 가격 | REB 전세가격지수 | 시군구 | 12월 값 |
| 거래 | 매매 거래건수 | 시군구 | 연간 SUM |
| 거래 | 전월세 거래건수 | 시군구 | 연간 SUM |

#### 시트3: 로우데이터 (유료 옵션)

해당 시군구 + 해당 기간의 건별 실거래 데이터. 전국 전체 덤프가 아닌 사용자 선택 조건 한정.

Phase 1에서는 미구현. Phase 2에서 유료 옵션으로 제공 예정.

항목 선정 및 시트 구성은 Phase 2 착수 시 확정한다.

#### 시트4: 참고자료

참고 지표: CPI, CSI, ESI (동일 기간, 전국 단위).

고지문: 데이터 출처, 면적 기준, 집계 방식, 결측 정책, "분양최고금액 기준" 명시.

하단 고지문: "본 자료는 공공데이터포털·한국은행·한국부동산원 API 제공 데이터 기준으로 작성되었으며, API 미제공 기간 또는 지역의 데이터는 누락될 수 있습니다."

## 7. 핵심 구성요소

### 7.1 데이터 레이어

fact_trade_raw, fact_indicator_month, dim_region, dim_complex. PostgreSQL 기반, raw SQL + view 중심 설계.

### 7.2 단지 마스터 (dim_complex)

PK: apt_seq. 1차 생성: fact_trade_raw에서 추출. 2차 보강: 건축물대장 API(좌표, 세대수, 공급면적). Phase 2 반경 검색 전 완료.

### 7.3 정렬 엔진 (핵심 IP)

동일 기간 강제 정렬. 월 기준 캘린더 테이블 기반 집계. 월별/연별 두 가지 집계 모드. 중앙값·건수 계산. 반기 지표 직전값 캐리. 기준금리 월말 유효값 적용. 결측 "-" 처리. 세부 명세는 ENGINE.md에서 정의한다.

### 7.4 엑셀 생성 엔진

Phase 1 산출물인 4시트 엑셀 파일을 생성한다. openpyxl 기반. 모든 시트에 단위, 기준, 집계 방식을 명시한다.
